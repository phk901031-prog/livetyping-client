# LiveTyping Online — 클라이언트 파일 구조

우리 앱을 **사람의 몸**에 비유하면 이렇게 생겼어요.

---

## 전체 구조 한눈에 보기

```
src/
├── main.tsx              🫀 심장 — 앱을 처음 뛰게 하는 곳
├── App.tsx               🧠 뇌 — 모든 판단과 명령을 내리는 중추
├── App.css               👕 옷 — 앱의 전체 외형(색상, 크기, 배치)
├── types.ts              🦴 뼈대 — 데이터의 모양을 정의하는 설계도
├── vite-env.d.ts         🪪 신분증 — TypeScript에게 "이런 것도 있어" 알려주는 파일
│
├── components/           👐 손발 — 사용자가 직접 보고 만지는 화면 조각들
│   ├── Lobby.tsx         🚪 현관문 — 방 만들기/참여하기 화면
│   ├── Header.tsx        🎩 모자 — 화면 맨 위: 방 코드, 회의 상태, 버튼들
│   ├── SegmentList.tsx   📋 노트 — 입력된 문장들이 쌓이는 목록
│   ├── InputArea.tsx     ✍️ 손 — 문장을 타이핑하는 입력창
│   ├── VideoPanel.tsx    📺 눈 — YouTube/로컬 영상·음성 재생기
│   └── MacroPanel.tsx    📎 주머니 — 상용구(자주 쓰는 말) 관리 패널
│
├── hooks/                🫁 내장기관 — 눈에 안 보이지만 핵심 기능을 돌리는 엔진들
│   ├── useTheme.ts       🌗 피부 — 다크/라이트 테마 + 글자 크기 조절
│   ├── useMacros.ts      🗂️ 기억력 — 상용구 목록 저장·불러오기·관리
│   └── useMedia.ts       🎬 시신경 — 영상/음성 재생 + 상대방과 미디어 동기화
│
└── utils/                🧰 도구상자 — 여기저기서 꺼내 쓰는 작은 도구들
    └── helpers.ts        🔧 만능칼 — 단어 삭제, YouTube URL 파싱, 키 조합 문자열 변환
```

## 각 파일 상세 설명

### `main.tsx` — 🫀 심장
앱이 처음 실행될 때 딱 한 번 뛰는 심장이에요.
HTML 파일의 `<div id="root">`에 React 앱(`<App />`)을 꽂아 넣어서 화면을 시작시켜요.

### `App.tsx` — 🧠 뇌
앱의 **두뇌**. 모든 정보가 여기로 모이고, 여기서 판단해서 각 부위에 명령을 내려요.
- 서버(Socket.io)와 통신하는 신경망 관리
- 세그먼트(문장) 생성·수정·순서 변경
- 키보드 입력 처리 (한글 IME 대응 포함)
- 각 컴포넌트에 필요한 데이터를 전달

### `App.css` — 👕 옷
앱의 **외형 전체**를 담당해요. 색상, 크기, 위치, 애니메이션 등 눈에 보이는 모든 스타일이 여기에 있어요.
CSS 변수(`--bg`, `--fg` 등)로 다크/라이트 테마를 한 번에 바꿀 수 있게 해놨어요.

### `types.ts` — 🦴 뼈대
데이터의 **골격**을 정의하는 설계도예요.
"세그먼트는 index, user, content, status를 가져야 해" 같은 규칙을 TypeScript 타입으로 적어놓은 거예요.
이게 있으면 코드 짤 때 오타나 빠뜨린 값을 미리 잡아줘요.

---

### `components/` — 👐 손발 (화면 조각들)

| 파일 | 비유 | 하는 일 |
|------|------|---------|
| **Lobby.tsx** | 🚪 현관문 | 앱 첫 화면. 이름 입력 → 방 만들기 or 코드 입력해서 참여하기 |
| **Header.tsx** | 🎩 모자 | 화면 맨 위 바. 방 코드 표시, 회의 시작/휴식/종료 버튼, 테마·폰트 설정 |
| **SegmentList.tsx** | 📋 노트 | 속기사들이 입력한 문장들이 쭉 나열되는 목록. 클릭하면 수정 가능 |
| **InputArea.tsx** | ✍️ 손 | 화면 맨 아래 입력창. 여기서 문장을 타이핑하고 Enter로 전송 |
| **VideoPanel.tsx** | 📺 눈 | YouTube URL 입력 또는 파일 불러오기 → 영상/음성 재생 + 배속 조절 |
| **MacroPanel.tsx** | 📎 주머니 | 상용구 관리. 자주 쓰는 말을 등록해두고 단축키로 빠르게 입력 |

---

### `hooks/` — 🫁 내장기관 (보이지 않는 엔진들)

React에서 **hook**은 "상태(state)와 로직을 묶어놓은 함수"예요.
컴포넌트가 손발이라면, hook은 그 손발을 움직이게 하는 근육이나 내장기관이에요.

| 파일 | 비유 | 하는 일 |
|------|------|---------|
| **useTheme.ts** | 🌗 피부 | 다크/라이트 모드 전환 + 글자 크기 조절. localStorage에 저장해서 새로고침해도 유지 |
| **useMacros.ts** | 🗂️ 기억력 | 상용구 목록을 localStorage에서 읽고 쓰고 관리. 추가/삭제/설정 변경 |
| **useMedia.ts** | 🎬 시신경 | YouTube 재생, 로컬 파일 재생, 배속 조절, 상대방에게 미디어 공유(동기화) |

---

### `utils/` — 🧰 도구상자

| 파일 | 비유 | 하는 일 |
|------|------|---------|
| **helpers.ts** | 🔧 만능칼 | 여러 곳에서 쓰이는 작은 함수 3개: ① 키보드 단축키 문자열 만들기 ② 커서 앞 단어 삭제 ③ YouTube URL에서 영상 ID 뽑기 |

## 데이터 흐름 (피가 도는 순서)

```
사용자 키보드 입력
       ↓
  InputArea (손) → App.tsx (뇌) → Socket.io 서버
       ↓                              ↓
  SegmentList (노트)          상대방 App.tsx (뇌)
       ↓                              ↓
    화면에 표시              상대방 SegmentList (노트)
```

1. **InputArea**(손)에서 타이핑 → Enter 누르면 **App.tsx**(뇌)에 전달
2. **App.tsx**(뇌)가 서버에 "새 문장이요!" 전송 + 화면에 바로 표시 (Optimistic Update)
3. 서버가 상대방에게 전달 → 상대방 **SegmentList**(노트)에도 표시
